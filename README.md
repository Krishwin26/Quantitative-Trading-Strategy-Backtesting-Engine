# Database-Driven Quantitative Trading Strategy Engine

A complete end-to-end framework combining Database Management Systems (DBMS) with Quantitative Finance. This project stores historical market data, generates trading signals using quantitative strategies (Mean Reversion), performs backtesting simulations, saves results to a SQL database, and evaluates trade performance.

## Setup Instructions

### Prerequisites
- Python 3.8+
- SQLite (Built-in with Python, used as default)
- (Optional) PostgreSQL or MySQL for production scalability.

### Installation

1. **Clone the repository or navigate to the project directory:**
   ```bash
   cd quant-trading-dbms
   ```

2. **Install dependencies:**
   ```bash
   pip install -r requirements.txt
   ```

3. **Run the Full Pipeline:**
   ```bash
   python main.py
   ```
   *Execution will automatically:*
   - Generate synthetic historical data for TSLA and AAPL.
   - Initialize the SQLite database `database/quant_trading.db`.
   - Calculate Z-Scores and generate BUY/SELL signals.
   - Backtest the strategy.
   - Compute key performance metrics (Sharpe Ratio, Max Drawdown).


## Database Architecture

The system utilizes a relational schema to ensure ACID compliance and standardized time-series querying.
- **`Stocks`**: Target equities.
- **`Price_Data`**: Daily OHLCV records.
- **`Trading_Signals`**: The explicit BUY/SELL recommendations generated by the strategy.
- **`Trades`**: Complete round-trip transactions including entry price, exit price, and Profit/Loss.
- **`Portfolio_Performance`**: Daily snapshots of the total mark-to-market portfolio value.

While the default implementation leverages SQLite via `SQLAlchemy` for zero-configuration testing, the schema `schema.sql` is strictly modeled for robust RDBMS (Postgres/MySQL) environments.

## Trading Strategy Logic

**Mean Reversion:**
The strategy theorizes that asset prices naturally return to their historical average over time.
1. Computes a **20-day Simple Moving Average (SMA)**.
2. Computes the **20-day Rolling Standard Deviation** to measure current volatility.
3. Derives the **Z-Score**: `(Current Close - SMA) / Standard Deviation`.
4. **Logic**:
    - **BUY** when `Z-Score < -2.0` (Statistically oversold / cheap).
    - **SELL** when `Z-Score > 2.0` (Statistically overbought / expensive).
